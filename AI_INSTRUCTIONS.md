# AI Collaboration Rules

Hello! If you are an AI assistant or a frontier model working on this codebase, you **MUST** adhere to the following strict rule:

## 1. Keep the AI History Updated
At the end of every session where you make significant code, architecture, or design changes, you are required to document your work. 

**Instructions:**
1. Navigate to the `ai_history/sessions/` directory.
2. Create a new markdown file named with the current date and a brief title (e.g., `YYYY-MM-DD_Feature_Name.md`).
3. Document the context, objectives, architectural decisions, and files modified.
4. Open `ai_history/index.md` and append a link to your new session log at the bottom of the "Session Log Index".
5. Update the "Key Architectural Pillars" section in `index.md` if your changes fundamentally altered how the application operates.

By following this rule, you ensure that the next AI model (or a future instantiation of yourself) has the complete context needed to continue seamlessly.
